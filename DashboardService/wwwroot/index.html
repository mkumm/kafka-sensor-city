How this works

What you're seeing

Each dot is one monitored intersection. The city has 5 districts, each with ~40 sensors spread across four road types: intersections, highways, parking areas, and pedestrian zones. Only intersection sensors appear here — the filter service upstream drops everything else before it reaches Kafka.

Intersection IDs

The ID encodes location: int-0105 means district 1, intersection 5. District number comes first (zero-padded to two digits), followed by the intersection sequence within that district. Districts are fixed at startup; intersections are discovered dynamically as their first event arrives.

When does a dot turn red?

Each sensor reading carries vehicle count, average speed, and road occupancy. A reading is flagged congested when occupancy > 75% and speed < 25 km/h at the same time. The aggregator maintains a rolling 30-second window per intersection. Once 3 or more congested readings land in that window, the intersection flips Congested. It returns Normal when the window clears.

Why does the map build up gradually?

The dashboard only receives state changes from Kafka, not every sensor reading. A dot appears the first time that intersection transitions state. Hover over any dot to see its current vehicle count, speed, occupancy, and the reason the last change was triggered.
